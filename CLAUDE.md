## Repo layout

- `spekta-app/` тАФ the actual application (Laravel). All commands below run from here.
- `spekta-docs/` тАФ product spec documents (PRD, REQUIREMENTS, BUSINESS_RULES, ARCHITECTURE, тАж) generated by Spekta itself. **They describe a target NestJS/Next.js architecture that is NOT the implementation** тАФ the real app is Laravel + Inertia. Use them for product intent (FR-xx / BR-xx codes), not code structure.
- `wireframe-spec-engine.html`, `hifi-design-spectra.html`, `analisa-spec-generator-tool.md` тАФ original design/analysis artifacts.

Code comments cite `FR-xx` (functional requirements), `BR-xx` (business rules), and `ADR-x` тАФ these resolve to `spekta-docs/REQUIREMENTS.md`, `BUSINESS_RULES.md`, and `ARCHITECTURE.md`.

Product language is **Bahasa Indonesia**: UI copy, LLM prompts, and most comments. Keep that convention.

## Commands (run in `spekta-app/`)

```bash
composer dev                        # serve + queue:listen + pail logs + vite, concurrently
php artisan test                    # full test suite (in-memory sqlite, stub LLM driver)
php artisan test --filter=MvpFlow   # single test class
vendor/bin/pint                     # PHP formatter
npm run lint                        # eslint --fix
npm run format                      # prettier on resources/
npm run build                       # vite production build
php artisan migrate                 # sqlite: database/database.sqlite
```

**The queue worker is required for document generation.** Runs stuck at "running" with an idle queue usually mean no worker is alive (`composer dev` includes one; standalone: `php artisan queue:listen --tries=3 --timeout=600`). Job timeout is 540s and must stay < listener timeout (600) < `retry_after` (630).

`.env*` files are blocked by tool permissions тАФ env reference lives in `env.spekta.example`. LLM driver auto-selects: `anthropic` if `ANTHROPIC_API_KEY` set, `openai` if `OPENAI_API_KEY`, otherwise deterministic `stub` (free, used by tests).

## Architecture

Laravel 13 monolith, Inertia 2 + React 19 + Tailwind 4 + Radix/shadcn frontend. One large page component per screen in `resources/js/pages/` (e.g. `wizard.tsx` ~1700 lines, `project.tsx` ~950). Routes are all in `routes/web.php`.

### Generation pipeline (the core of the product)

1. **Wizard flow** (FR-01..07, `WizardController` + `pages/wizard.tsx`): input тЖТ AI understanding тЖТ adaptive interview тЖТ structure (phases/features tree) тЖТ stack choice тЖТ generate.
2. **`Services/SpecEngine.php`** тАФ single gateway for ALL LLM calls (ADR-3). Drivers: `anthropic`, `openai`-compatible, `stub`. Node-class routing (`reasoning`/`standard`/`economy`) maps to models via `config/spekta.php`. Output length uses a *soft cap* in the prompt plus `max_tokens` as safety net; `guardTruncation()` throws `LlmTruncated` when output hits `max_tokens`.
3. **`Services/GenerationPipeline.php`** тАФ builds a `GenerationRun` with `GenerationNode`s from the doc dependency DAG, topo-sorts, dispatches a sequential job chain (single-worker MVP, no parallel branches).
4. **`Jobs/GenerateDocumentJob.php`** тАФ one job per document. Reads upstream docs' current versions as context, streams progress into cache key `genstream:{runId}` (polled by the wizard via `projects/{project}/run-status`). `LlmTruncated` fails fast тАФ deterministic, retrying would burn the same tokens. Resume (`generate/resume`) skips `done` nodes (BR-11).
5. **`Services/SpecHealthValidator.php`** тАФ cross-document consistency checks producing `HealthFinding`s and the project health score.

The DAG and doc sets are config, not code: `config/spekta.php` `doc_pipeline` (doc тЖТ upstream deps) and `doc_sets` (complexity 1тАУ5 тЖТ which docs get generated). Adding a doc type means touching that config plus SpecEngine's per-doc prompt.

### `config/spekta.php` is the business-rule source of truth

Plans/quotas (BR-01), top-up pricing, the estimate math (`role_split` is the ONLY source for Estimator, RabExporter, ChangeRequestService, and rate-card UI тАФ must sum to 1.0), work-mode multipliers, and LLM model routing all live here.

### Other subsystems

- **Documents**: `Document` + immutable `DocumentVersion`s; restore creates a new version.
- **Estimator** (FR-13/14): `Services/Estimator.php` from structure man-days; per-line overrides; exports via `Exporter`/`RabExporter`/`ProposalGenerator` (dompdf, phpword, phpspreadsheet).
- **Client portal** (FR-17..19): unauthenticated `portal/{token}` routes, OTP-gated, comments/approvals/change-requests. Change requests flow through `ChangeRequestService`.
- **Billing** (FR-23): Midtrans Snap; webhook `midtrans/notify` is auth/CSRF-free, sha512 signature verified in `MidtransBilling`.
- **Multi-tenancy**: `Workspace`/`WorkspaceMember`; user's current workspace via `User::currentWorkspace()`.

### Testing

Feature tests in `tests/Feature` cover whole flows (`MvpFlowTest` is the end-to-end wizardтЖТgenerateтЖТexport path). They run against the stub LLM driver тАФ no network, no cost. Prefer extending these over unit tests; the stub driver makes full-pipeline assertions cheap.
